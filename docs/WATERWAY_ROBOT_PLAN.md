# 用水路清掃ロボット計画

更新日: 2026-05-18

## 前提

このプロジェクトは、40cm級のタイヤ/キャタピラ式ロボットを用水路内で走らせ、ゴミ、泥、落ち葉、枝、詰まりを見つけながら清掃を続けるためのローカル頭脳を目指します。

現時点では実モータや清掃機構は接続しません。先に、止まれる、低速操作を提案できる、危険を表示できる、作業状態を残せる、という操作卓と安全境界を作ります。

## 役割分担で見た優先順位

| 役割 | 重点 | 直近の実装 |
|---|---|---|
| 企画役 | まず止まれること、次に低速で進めること | 緊急停止、Safety Gate、ダミーActuator |
| 実装役 | 会話ループと実機制御を混ぜないこと | `src/robot/` に制御境界を追加 |
| 検証役 | モデルなしでも安全系をテストできること | ユニットテストとWeb APIスモーク |
| ユーザー役 | 現場では会話より操作と状態が主役 | Web UIに操作卓、ミッション、AI認識根拠を表示 |

## 今回入れた制御境界

```text
Operator / Voice
  ↓
Web Console / Agent
  ↓
WorldState
  観察メモから、水路境界、ゴミ、泥、枝、詰まり、危険を抽出
  ↓
RobotPlanner
  会話返答とは別に ActionPlan を作る
  ↓
SafetyGate
  緊急停止、通信、姿勢、バッテリー、視覚警告を確認
  ↓
DummyActuator
  実モータの代わりに状態更新とログだけ行う
```

## Web操作卓

追加した操作:

- 緊急停止、緊急停止解除
- 前進、後退、左右旋回、停止
- 清掃ON/OFF、ライト切替
- ミッション開始、一時停止、再開、完了
- AI観察メモ、危険、ActionPlan表示

これらはすべてローカルAPIで完結します。現時点の操作はシミュレーションで、実機モータへは送信しません。

## 次の実装順

1. Web UIにイベント履歴を時系列表示する
2. `WorldState` を観察メモだけでなく、将来のToF/IMU/電流センサー入力からも更新する
3. `SafetyGate` に最大連続走行時間、最大速度、清掃モータ過負荷、浸水検知を追加する
4. 実機前に、JSONログだけのリプレイテストを作る
5. マイコン接続用の `ActuatorAdapter` をダミーから差し替え可能にする
6. 低速1m直進、障害物停止、清掃ON/OFFの実機受入テストを作る
